# LifecyclePolicy.IsEligibleForRemoval method

Returns `true` when a Deprecated feature is eligible for removal — i.e. its minimum window has elapsed as of *now* (R-27.2-c, R-27.2-l).

```csharp
public static bool IsEligibleForRemoval(DateOnly deprecatedSince, DateOnly now, 
    bool expedited = false)
```

| parameter | description |
| --- | --- |
| deprecatedSince | The date the feature first became Deprecated. |
| now | The date to test eligibility against (usually today). |
| expedited | When `true`, applies the 90-day minimum instead of 12 months. |

## Return Value

`true` when *now* is on or after the earliest-removal date.

## Remarks

Eligibility is a necessary condition for removal, not a mandate: a feature MAY remain Deprecated indefinitely (R-27.2-d). For the standard window the earliest-removal date is *deprecatedSince* plus 12 calendar months (with end-of-month clamping); for the expedited window it is *deprecatedSince* plus exactly 90 days. The comparison is inclusive — a feature is eligible on the boundary date itself.
