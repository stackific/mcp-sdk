## Remarks

The rules enforced here are:

* Active → Removed is FORBIDDEN; a feature MUST pass through Deprecated first (R-27.2-b).
* Removed is terminal — no transition out of it is permitted (R-27.2).
* A same-state "transition" is not a transition and is rejected (R-27.1-e).
* The standard minimum window is 12 calendar months (R-27.2-c).
* The security-expedited minimum window is 90 days (R-27.2-l).
* A Deprecated feature MAY be restored to Active; on re-deprecation the window is measured afresh (R-27.2-n, R-27.2-p).

Date arithmetic uses the calendar-only DateOnly type so it is deterministic and free of host-timezone distortion. Callers supply the "now" date explicitly — the policy never reads the wall clock — mirroring the TypeScript functions that take dates as parameters.
