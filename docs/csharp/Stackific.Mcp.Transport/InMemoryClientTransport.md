# InMemoryClientTransport class

A client transport that bridges directly to an in-process [`IMcpRequestHandler`](./IMcpRequestHandler.md) (an `McpServer`) with no HTTP or serialization round-trip. It mirrors the Streamable HTTP request shape — each request gets its own logical response stream over which interim notifications are delivered before the final response (spec §9.6) — which makes it ideal for tests and for embedding a server in the same process. Frames are still offered to the wire taps.
