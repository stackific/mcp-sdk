# InMemoryByteChannelTransport class

The S12 in-memory reference byte-channel transport (spec §7.1–§7.6) — a fully in-process [`IByteChannelTransport`](./IByteChannelTransport.md) used to demonstrate and test that the §7.2 guarantees can be met by a conforming transport. The C# counterpart of the TypeScript `InMemoryTransport` in `transport/in-memory.ts`.

## Remarks

This is itself a custom transport in the §7.3 sense: it preserves the JSON-RPC message format, the exchange patterns, and the per-request metadata model, and upholds every §7.2 guarantee — but it is NOT one of the two transports the specification defines (stdio is S13; Streamable HTTP is S14/S15). It is distinct from [`InMemoryClientTransport`](./InMemoryClientTransport.md), which is a convenience RPC bridge to an in-process handler rather than an observable byte channel.

To make the framing, UTF-8, and integrity guarantees real rather than assumed, the pair carries bytes: each [`Send`](./InMemoryByteChannelTransport/Send.md) frames the message with [`NewlineFramer`](./NewlineFramer.md) and the peer recovers it with the same framing plus [`TryDecode`](./MessageUnit/TryDecode.md) (UTF-8 + single-JSON-value validation). Delivery is synchronous for test determinism.

Cancellation (§7.3). A custom byte-channel transport carries NO per-request cancellation token at the channel layer — the channel frames opaque messages and is unaware of request/response correlation. Cancellation is therefore connection-scoped: closing the channel ([`Disconnect`](./InMemoryByteChannelTransport/Disconnect.md)) aborts all outstanding work in one shot, and each side observes the teardown through its [`OnClose`](./InMemoryByteChannelTransport/OnClose.md) subscription (after which [`Send`](./InMemoryByteChannelTransport/Send.md) throws a [`TransportError`](./TransportError.md)). Per-request cancellation of a single in-flight request — without tearing down the connection — is expressed at the protocol layer instead, via a `notifications/cancelled` message referencing the request id (§15.2), which a server host (for example [`StdioServerHost`](../Stackific.Mcp.Server/StdioServerHost.md)) honors by aborting that request and suppressing its response (§8.3 R-8.3-g).
