## Remarks

Enforces, in order:

1. UTF-8. The bytes MUST be well-formed UTF-8; an invalid unit is rejected with a [`TransportError`](../TransportError.md), never silently substituted (R-7.6-a, R-7.6-b, R-7.6-c).
2. Single JSON value. The text MUST parse as exactly one JSON value; trailing or multiple values are rejected (R-7.1-b, R-7.6-b).
3. Well-formed message. The value MUST classify as one of the three [`JsonRpcMessage`](../../Stackific.Mcp.JsonRpc/JsonRpcMessage.md) kinds; otherwise rejected.

The method never returns a substituted or partial message — every failure is an observable throw (R-7.2-q, R-7.6-c).
