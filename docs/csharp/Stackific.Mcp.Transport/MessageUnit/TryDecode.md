# MessageUnit.TryDecode method

Non-throwing variant of [`Decode`](./Decode.md): returns `false` with the [`TransportError`](../TransportError.md) in *error* instead of throwing (spec §7.6). Mirrors TypeScript `tryDecodeMessageUnit`.

```csharp
public static bool TryDecode(ReadOnlySpan<byte> bytes, out JsonRpcMessage? message, 
    out TransportError? error)
```

| parameter | description |
| --- | --- |
| bytes | The framed unit's bytes. |
| message | On success, the decoded message; otherwise `null`. |
| error | On failure, the decode error; otherwise `null`. |

## Return Value

`true` when the unit decoded successfully.

## Remarks

The failure is still observable (it is returned, not swallowed) so the no-silent-drop rule (R-7.6-c) holds.
