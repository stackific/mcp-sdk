# RequestCorrelator.Issue method

Registers *id* as outstanding and returns a Task that completes when a matching response is delivered or the request is failed (spec §7.2).

```csharp
public Task<JsonRpcMessage> Issue(RequestId id)
```

| parameter | description |
| --- | --- |
| id | The id of the request being sent. |

## Return Value

A Task that resolves with the matching response, or faults with a [`TransportError`](../TransportError.md).

## Exceptions

| exception | condition |
| --- | --- |
| InvalidOperationException | Synchronously when *id* is already outstanding — a sender MUST NOT reuse the id of an unanswered request (R-7.2-j). |

## Remarks

Concurrency: calling `Issue` again before the first completes is allowed and expected — the transport need not await one response before issuing another (R-7.2-i, R-7.2-k, R-7.2-l). The returned Task uses RunContinuationsAsynchronously so a continuation never runs inline on the thread that calls [`Deliver`](./Deliver.md) / [`FailAll`](./FailAll.md), preventing re-entrancy into the transport's receive loop.
