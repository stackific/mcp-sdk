## Remarks

Each message is its UTF-8 JSON serialization followed by a single `\n`. A receiver recovers messages by splitting on `\n` without parsing the body (R-7.2-c, R-7.2-d). This is the framing a custom transport over a reliable bidirectional byte stream (Unix socket, TCP) SHOULD reuse rather than defining a new one (R-7.3-e); the stdio transport (S13) is this framing plus process-lifecycle rules.
