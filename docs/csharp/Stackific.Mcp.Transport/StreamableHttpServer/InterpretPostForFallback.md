| parameter | description |

## Return Value

The fallback decision.

## Remarks

On a `400`, the client SHOULD inspect the body before falling back, because a modern server returns `400` for `-32004`/`-32003`/`-32001`. A recognized revision error means retry (using `error.data.supported` when present), never fall back. An empty/unrecognized body on a `400`/`404`/`405` means the client SHOULD probe for the legacy transport.
