# TransportContract.RequestCarriesMetaEnvelope method

Returns `true` when a request carries the inline `_meta` envelope with the three reserved `io.modelcontextprotocol/*` keys (spec §7.4; R-7.4-d, R-7.4-f). Mirrors TypeScript `requestCarriesMetaEnvelope`.

```csharp
public static bool RequestCarriesMetaEnvelope(JsonRpcMessage? request)
```

| parameter | description |
| --- | --- |
| request | The candidate request message. |

## Return Value

`true` when a valid three-key envelope is present on `params._meta`.

## Remarks

The inline envelope is REQUIRED regardless of transport; the message body is the source of truth. A transport MAY additionally mirror these fields into transport-level metadata (see [`ExtractEnvelopeForMirroring`](./ExtractEnvelopeForMirroring.md)), but that mirror is never a substitute for the inline envelope.
