# TransportContract.DeriveRequestContext method (2 of 2)

Derives the per-request context (protocol version, client identity, client capabilities) solely from the request's own `_meta`, never from the connection or any prior request (spec §7.6; R-7.6-e, R-7.6-f). Mirrors TypeScript `deriveRequestContext`.

```csharp
public static RequestContext? DeriveRequestContext(JsonRpcMessage? request)
```

| parameter | description |
| --- | --- |
| request | The candidate request message. |

## Return Value

The derived context, or `null` when no valid envelope is present.

## Remarks

Returns `null` when the message is not a request, or does not carry a valid envelope; the server then has no basis to process it (and MUST NOT infer one from earlier requests). Two requests on the same connection with different envelopes yield two independent contexts — the connection contributes nothing.
