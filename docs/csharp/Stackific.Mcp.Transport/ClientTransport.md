# ClientTransport class

The client-side transport abstraction: it sends a single JSON-RPC request or notification and, for a request, returns the final response — delivering any interim request-scoped notifications through [`OnNotification`](./RequestOptions/OnNotification.md). Concrete transports are the in-memory bridge ([`InMemoryClientTransport`](./InMemoryClientTransport.md)) and Streamable HTTP. Every outbound and inbound frame is offered to [`OnSend`](./ClientTransport/OnSend.md)/[`OnReceive`](./ClientTransport/OnReceive.md) so a host can tap the wire (§9).
