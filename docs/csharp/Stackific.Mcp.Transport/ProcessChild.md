## Remarks

The process MUST be started with redirected stdin/stdout/stderr. This adapter bridges the process's redirected streams to the push-based [`IByteSource`](./IByteSource.md)/[`IByteSink`](./IByteSink.md) contract: it pumps stdout and stderr on background reader loops and writes stdin synchronously. On POSIX, Terminate maps to a graceful kill (entire process tree is not killed) and Force to `Process.Kill(entireProcessTree: true)`; on Windows both map to a forced kill, which is the only available mechanism.

Use [`Launch`](./ProcessChild/Launch.md) to start a redirected process and wrap it, mirroring the typical stdio client launch. This type is deliberately thin so the deterministic tests can exercise the transport via in-memory pipes instead.
