## Remarks

A JSON-RPC error response (an `error` object inside a delivered message) is a normal, fully delivered protocol message reporting that a request failed at the protocol/application layer. A [`TransportError`](./TransportError.md) instead signals that the channel could not carry a message, that a received unit was malformed at the encoding/framing level, or that the connection was lost — i.e. an observable transport-level failure (R-7.2-q, R-7.2-r, R-7.5-i, R-7.5-j, R-7.6-b).

This is the C# counterpart of the TypeScript `TransportError`; it carries the same stable, machine-readable [`Code`](./TransportError/Code.md) (`"TRANSPORT_ERROR"`) so callers can branch on a channel failure programmatically without inspecting the message text.
