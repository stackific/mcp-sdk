# TransportCloseInfo constructor

Why a transport channel became unusable, surfaced to `OnClose` handlers (spec §7.2/§7.5).

```csharp
public TransportCloseInfo(bool Clean, string? Reason = null)
```

| parameter | description |
| --- | --- |
| Clean | `true` for an orderly close; `false` for an abrupt disconnection. |
| Reason | An optional human-readable explanation, or `null`. |

## Remarks

[`Clean`](./Clean.md)`true` is an orderly shutdown each side had the opportunity to observe (R-7.2-t). [`Clean`](./Clean.md)`false` is an abrupt disconnection — the channel dropped without an orderly close — which a transport MUST still make observable (R-7.5-a, R-7.5-b).
