# StreamableHttpClientTransport class

The client-side Streamable HTTP transport (spec §9): each request or notification is its own HTTP POST to the MCP endpoint, carrying the required headers (§9.3) and routing headers (§9.4). A request response is read as either a single JSON object (§9.6.1) or a server-sent event stream that delivers request-scoped notifications before the final response (§9.6.2). An optional token provider supplies the bearer credential for authorized resources (§23).
