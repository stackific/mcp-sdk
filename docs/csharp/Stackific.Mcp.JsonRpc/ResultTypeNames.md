## Remarks

Additional values MAY exist only when introduced via the extension mechanism (§24). The two values here are the complete spec-defined set used by [`IsKnownResultType`](./Payload/IsKnownResultType.md) and [`InterpretResultType`](./Payload/InterpretResultType.md) to decide whether a received `resultType` is recognized. The extension `"task"` value, used by the Tasks extension, is intentionally not part of this core set, mirroring `payload.ts`.
