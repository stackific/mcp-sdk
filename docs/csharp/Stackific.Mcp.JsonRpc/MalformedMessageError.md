## Remarks

This is the foundation-layer analogue of the TypeScript `MalformedMessageError`. It carries a stable, machine-readable [`Code`](./MalformedMessageError/Code.md) (`"MALFORMED_MESSAGE"`) so callers can branch on the rejection programmatically without depending on a numeric JSON-RPC code.

Per R-3.4-f, malformed notifications are silently discarded — callers MUST decide, based on the message kind, whether to surface this error toward the sender or to drop it. The runtime transport layer maps wire-level framing failures to [`McpError`](./McpError.md) (with codes −32600 / −32700); this type exists for the portable, kind-agnostic classification surface.
