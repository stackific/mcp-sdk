# Payload.InterpretResultType method

Interprets the `resultType` field of a received result object, applying both normative receiver rules from §3.6.

```csharp
public static ResultTypeInterpretation InterpretResultType(JsonObject result)
```

| parameter | description |
| --- | --- |
| result | The raw result object received from the wire. |

## Return Value

A [`ResultTypeInterpretation`](../ResultTypeInterpretation.md) whose [`Recognized`](../ResultTypeInterpretation/Recognized.md) flag tells the caller whether to read the rest of the result or treat the response as an error.

## Remarks

* R-3.6-i: an absent or `null``resultType` is treated as `"complete"` (interop fallback for servers that omit the field).
* R-3.6-f: an unrecognized value yields [`Recognized`](../ResultTypeInterpretation/Recognized.md) = `false`, signalling that the whole response must be treated as an error.
* R-3.6-g: when [`Recognized`](../ResultTypeInterpretation/Recognized.md) is `false`, callers MUST NOT read other members.

A non-null, non-string `resultType` (for example a number) can never be one of the two spec-defined string values, so it is reported as its JSON text form and treated as unrecognized — the intended outcome for a malformed discriminator.
