# Framing.IdEchoMatches method

Returns `true` when *responseId* is a correct echo of *requestId* — the same JSON type (string ↔ string, number ↔ number) and the same value, with NO type coercion. (R-3.2-e, R-3.2-f, R-3.2-g)

```csharp
public static bool IdEchoMatches(RequestId requestId, RequestId responseId)
```

| parameter | description |
| --- | --- |
| requestId | The id of the originating request. |
| responseId | The id carried by the response. |

## Return Value

`true` when the two ids match exactly in type and value.

## Remarks

[`RequestId`](../RequestId.md) equality is already type-fidelity-correct — a number `7` and a string `"7"` never compare equal — so this is a thin, intention-revealing wrapper that names the echo-validation rule and mirrors the TypeScript `idEchoMatches` helper.
