## Remarks

Per R-3.2-c a sender MUST NOT reuse an identifier while the original request is still awaiting a response; per R-3.2-d all outstanding ids from a single sender on a single connection MUST be unique.

A string id and a number id with the same textual representation are kept distinct because they are different JSON types (R-3.2-f, R-3.2-g): `"1"` and `1` are different ids. This is enforced by [`RequestId`](./RequestId.md)'s own equality and hashing, so the tracker can key directly on the [`RequestId`](./RequestId.md) value.
