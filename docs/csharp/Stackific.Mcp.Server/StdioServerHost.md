# StdioServerHost class

Serves an [`McpServer`](./McpServer.md) over a byte-channel transport — the stdio transport in practice (spec §8). It reads inbound client requests/notifications from the transport, dispatches each through [`HandleRequestAsync`](./McpServer/HandleRequestAsync.md), and writes the responses and request-scoped notifications back over the same channel.

## Remarks

Statelessness (§7.6): there is no session. Each request is self-contained and dispatched on its own.

No server→client request channel (§8.3 / §9.6.2 R-9.6.2-d). A server MUST NOT write a JSON-RPC request to its output stream. The host therefore never issues server-initiated requests; the supported mechanism for soliciting client input is the §11 `input_required` retry loop (via [`ElicitInputAsync`](./ToolContext/ElicitInputAsync.md) / [`CreateMessageAsync`](./ToolContext/CreateMessageAsync.md) / [`ListRootsAsync`](./ToolContext/ListRootsAsync.md)). An inbound JSON-RPC response is therefore unsolicited and ignored (§7.4). A `notifications/cancelled` aborts the matching in-flight request and suppresses its response (§8.3 R-8.3-g).
