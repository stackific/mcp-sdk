# UiHelpers.UiToolResult method

Builds a tool result that launches an MCP App (spec §26.3/§26.4): a text fallback block plus an embedded `ui://` resource carrying the app HTML, with the tool's `_meta.ui` declaring the resource URI (and, when supplied, the visibility audiences).

```csharp
public static CallToolResult UiToolResult(string resourceUri, string html, string? text = null, 
    IReadOnlyList<UiVisibility>? visibility = null)
```

| parameter | description |
| --- | --- |
| resourceUri | The `ui://` URI identifying the app resource. |
| html | The app HTML (served as `text/html;profile=mcp-app`). |
| text | An optional text fallback for hosts that do not render UI. |
| visibility | OPTIONAL invoking audiences ([`UiVisibility`](../../Stackific.Mcp.Protocol/UiVisibility.md)); when omitted the host treats the declaration as `["model","app"]` and the key carries only `resourceUri` (§26.3). |

## Return Value

The tool result.

## Exceptions

| exception | condition |
| --- | --- |
| ArgumentException | When *resourceUri* is not a `ui://` URI (R-26.3-b). |

## Remarks

The *resourceUri* MUST use the `ui://` scheme — a non-`ui://` value is a non-conformant UI association and is rejected (spec §26.3, R-26.3-b). When *visibility* is supplied it is written into the `_meta.ui.visibility` array; omitted, the host treats the declaration as `["model","app"]` (spec §26.3, R-26.3-d).
