# AuthGates class

Server-side authorization glue (spec §23): turns a token-validation callback into an [`IMcpAuthGate`](../Stackific.Mcp.Transport/IMcpAuthGate.md) for the Streamable HTTP adapter — emitting the `401` + `WWW-Authenticate` challenge (carrying `resource_metadata` and, when known, `scope`) and the `403``insufficient_scope` challenge the spec requires — and builds the RFC 9728 protected-resource metadata document. The C# counterpart of ts-sdk's `server/auth.ts`.

## Remarks

The gate's per-request state machine (missing/invalid/expired → `401`; wrong audience → `401`; under-scoped → `403`) and the challenge construction are delegated to the §23.8 protocol primitives ([`AccessTokenUsage`](../Stackific.Mcp.Protocol/AccessTokenUsage.md), [`WwwAuthenticate`](../Stackific.Mcp.Protocol/WwwAuthenticate.md)): the Bearer scheme is matched case-insensitively (RFC 7235), the audience is compared by canonical resource identity (string OR array, uppercase scheme/host and trailing-slash tolerant), and the challenge values are escaped per RFC 7235.
