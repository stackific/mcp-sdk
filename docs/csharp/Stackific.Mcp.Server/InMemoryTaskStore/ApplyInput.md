# InMemoryTaskStore.ApplyInput method

Binds the responses supplied via `tasks/update` to a task's outstanding input requests and moves the task back to Working (§25.8). Mirrors the TypeScript `applyInput`: it accepts ONLY responses keyed by a currently-outstanding request — stale keys (no longer outstanding) are dropped per R-25.8-h — and tolerates a partial set (R-25.8-g), so a caller MAY answer some requests now and the rest later. The bound responses are accumulated so a re-entrant tool run (which re-solicits from scratch each round) can replay them.

## Remarks

The caller (the server dispatcher) is responsible for the protocol-level validation of the responses — outstanding-key check ([`ValidateInputResponseKeys`](../../Stackific.Mcp.Protocol/MultiRoundTrip/ValidateInputResponseKeys.md)), kind-correlation ([`ValidateInputResponseKinds`](../../Stackific.Mcp.Protocol/MultiRoundTrip/ValidateInputResponseKinds.md)), and partial-response detection ([`ComputeMissingInputResponseKeys`](../../Stackific.Mcp.Protocol/MultiRoundTrip/ComputeMissingInputResponseKeys.md)). This method performs the state mutation only; it discards any response whose key is not currently outstanding.
