# InMemoryTaskStore.Get method

Returns the detailed state of a task for `tasks/get` (§25.7). The returned [`DetailedTask`](../../Stackific.Mcp.Protocol/DetailedTask.md) carries the status-specific inline payload (§25.5, R-25.5-d): [`Result`](../../Stackific.Mcp.Protocol/DetailedTask/Result.md) when `completed`, [`Error`](../../Stackific.Mcp.Protocol/DetailedTask/Error.md) when `failed`, and [`InputRequests`](../../Stackific.Mcp.Protocol/DetailedTask/InputRequests.md) when `input_required`.

## Remarks

Expiry (§25.6, R-25.6-f/g, R-25.11-d): before the lookup the store sweeps any task whose non-null `ttlMs` has elapsed (per [`IsTaskExpired`](../../Stackific.Mcp.Protocol/Tasks/IsTaskExpired.md)). A query for an expired (and thus discarded) task is therefore indistinguishable from a query for one that never existed — both answer with the `-32602` not-found error. The expiry boundary uses the helper's elapsed-or-equal (`now - created >= ttl`) rule.
