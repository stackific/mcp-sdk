# InMemoryTaskStore.RequestInput method

Moves a non-terminal task into InputRequired, recording the outstanding input solicitations the client must satisfy via `tasks/update` (§25.5, §25.8). The *inputRequests* are surfaced inline on the task's `inputRequests` and cleared once the task leaves the input-required state. A terminal task is immutable and ignored.
