# ConformanceRequirements.ServerRequestDisposition record

The disposition a conformant server reaches for an incoming request after the ordered §29.2 checks (spec §29.2). On a revision failure, [`Code`](./ConformanceRequirements.ServerRequestDisposition/Code.md) is `-32004` and [`Supported`](./ConformanceRequirements.ServerRequestDisposition/Supported.md) / [`Requested`](./ConformanceRequirements.ServerRequestDisposition/Requested.md) are populated; on an envelope failure, [`Code`](./ConformanceRequirements.ServerRequestDisposition/Code.md) is `-32602` with a [`Message`](./ConformanceRequirements.ServerRequestDisposition/Message.md); on a capability failure, [`Code`](./ConformanceRequirements.ServerRequestDisposition/Code.md) is `-32003` with [`RequiredCapabilities`](./ConformanceRequirements.ServerRequestDisposition/RequiredCapabilities.md); on a gating refusal, none of these are set.
