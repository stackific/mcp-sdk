# ElicitationForm.HandleElicitationComplete method

Decides how a client should react to an incoming elicitation-complete notification, enforcing the §20.6 ignore rule (spec §20.6, R-20.6-d, R-20.6-e). A client MUST ignore a notification whose `elicitationId` is unknown or already completed (R-20.6-d); for a still-pending id it MAY proceed to auto-retry, update its UI, or otherwise continue (R-20.6-e). Independently, a client SHOULD provide manual retry/cancel controls in case it never arrives (R-20.6-f) — a UI concern outside this pure decision.
