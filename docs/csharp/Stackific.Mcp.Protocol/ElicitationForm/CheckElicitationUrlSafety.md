# ElicitationForm.CheckElicitationUrlSafety method

Checks a server-constructed elicitation URL against the §20.7 safe-construction rules: it MUST NOT carry sensitive end-user info, MUST NOT be pre-authenticated to a protected resource, and SHOULD use HTTPS outside development (spec §20.7, R-20.7-p, R-20.7-q, R-20.7-s).

```csharp
public static ElicitationUrlSafety CheckElicitationUrlSafety(string? url, 
    bool allowInsecure = false)
```

| parameter | description |
| --- | --- |
| url | The elicitation URL the server intends to send. |
| allowInsecure | `true` permits non-HTTPS (development only). |

## Return Value

The safety outcome.

## Remarks

Heuristics flag credential/PII-looking query parameters and embedded userinfo (`user:pass@host`), and (outside *allowInsecure*) any non-`https:` scheme. This is a guard to catch obvious mistakes, not a guarantee of safety.
