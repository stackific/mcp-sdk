# ElicitationForm.ValidateElicitResult method

Validates a returned `ElicitResult` against the §20.5 action/content rules for the mode it answers (spec §20.5, R-20.5-a, R-20.5-b, R-20.5-c).

```csharp
public static ElicitResultValidation ValidateElicitResult(ElicitResult? result, string mode, 
    JsonNode? requestedSchema = null)
```

| parameter | description |
| --- | --- |
| result | The `ElicitResult` returned by the client. |
| mode | The mode of the originating request (`"form"` or `"url"`). |
| requestedSchema | The form-mode `requestedSchema` (used only to check `content` conformance on a form-mode accept). |

## Return Value

The validation outcome.

## Remarks

Enforced:

* `action` is REQUIRED and exactly one of `accept`/`decline`/`cancel`;
* `content` is permitted ONLY when `action == "accept"` AND the mode is form; a URL-mode accept, a decline, or a cancel carrying `content` is malformed (the URL-mode case is a credential-leak vector);
* when `content` is present (form-mode accept), it conforms to *requestedSchema* per [`ValidateElicitContent`](./ValidateElicitContent.md) — supply the schema to enable this check.

Content values are additionally checked to be the permitted types (string &#x7C; number &#x7C; boolean &#x7C; string[]); a disallowed type (object, null, mixed array) makes the result malformed.
