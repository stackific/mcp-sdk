# ElicitationForm.BuildUrlConsentPresentation method

Builds the consent-presentation data a client MUST show before opening a URL-mode elicitation URL: the full URL and a clearly-highlighted target host, plus warnings about Punycode / ambiguous URIs (spec §20.7, R-20.7-v, R-20.7-x).

```csharp
public static UrlConsentPresentation BuildUrlConsentPresentation(string url)
```

| parameter | description |
| --- | --- |
| url | The URL-mode elicitation URL. |

## Return Value

The consent presentation.

## Exceptions

| exception | condition |
| --- | --- |
| ArgumentException | When *url* is not a valid absolute URL. |

## Remarks

This produces the data a UI binds to; it does NOT open the URL or prefetch it (a client MUST NOT prefetch — R-20.7-t — and MUST NOT open without consent — R-20.7-u). The host is exposed separately so the UI can highlight it to defend against subdomain spoofing, and a Punycode host raises a warning.
