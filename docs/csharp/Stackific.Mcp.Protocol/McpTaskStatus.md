# McpTaskStatus enumeration

The lifecycle state of a task (spec §25.5). A task is created non-terminal (typically Working) and moves among Working and InputRequired until it reaches one of the terminal states Completed, Failed, or Cancelled, after which its status and inline payload are immutable.

## Values

| name | value | description |
| --- | --- | --- |
| Working | `0` | The operation is in progress (non-terminal) (§25.5). |
| InputRequired | `1` | The server requires client input before it can continue (non-terminal); the outstanding requests are surfaced in `inputRequests` and resolved via `tasks/update` (§25.5, §25.8). |
| Completed | `2` | The operation finished successfully; the underlying result is carried inline in `result` (terminal) (§25.5). |
| Failed | `3` | A JSON-RPC error occurred during execution; the error is carried inline in `error` (terminal) (§25.5). |
| Cancelled | `4` | The operation ended in response to a cancellation request (terminal) (§25.5). |
