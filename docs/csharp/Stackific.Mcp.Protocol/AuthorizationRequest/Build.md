# AuthorizationRequest.Build method

Builds the authorization-request query parameters for Step 2, fixing `response_type=code`, `code_challenge_method=S256`, the `code_challenge` and `state` from the Step-1 record, and the REQUIRED `resource` parameter (R-23.5-d, R-23.5-e, R-23.5-g, R-23.5-i, R-23.5-j, R-23.6-b).

```csharp
public static AuthorizationRequestParams Build(string clientId, string redirectUri, 
    string resource, AuthorizationFlowRecord record, string? scope = null, 
    IReadOnlyList<string>? confirmPkceFrom = null)
```

| parameter | description |
| --- | --- |
| clientId | The client identifier. |
| redirectUri | MUST match one registered for the client (R-23.5-e). |
| resource | The canonical resource identifier of the target MCP server (R-23.5-j). |
| record | The Step-1 record carrying the PKCE challenge and `state`. |
| scope | OPTIONAL pre-resolved `scope`. |
| confirmPkceFrom | OPTIONAL AS `code_challenge_methods_supported` to gate on PKCE support (R-28.5-k). |

## Return Value

The authorization-request parameters.

## Exceptions

| exception | condition |
| --- | --- |
| [PkceSupportException](../PkceSupportException.md) | When *confirmPkceFrom* is supplied and PKCE support cannot be confirmed. |

## Remarks

When *confirmPkceFrom* is supplied, the builder verifies PKCE `S256` support and refuses (throws [`PkceSupportException`](../PkceSupportException.md)) if it cannot be confirmed — enforcing §28.5 (R-28.5-k). Callers that omit it MUST call [`AssertConfirmed`](../PkceSupport/AssertConfirmed.md) themselves.
