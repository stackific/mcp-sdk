# ClientCapabilities.HasExtension method

Returns `true` if the client VALIDLY advertised the extension *identifier* (§6.5). Routed through [`IsAdvertised`](../Extensions/IsAdvertised.md) so an entry that is present but malformed — a `null` or non-object settings value — is NOT treated as advertised (R-6.5-j). A bare ContainsKey check would wrongly count a `null`-valued key.

```csharp
public bool HasExtension(string identifier)
```

| parameter | description |

| identifier | The extension identifier, for example `io.modelcontextprotocol/tasks`. |

## Return Value

`true` when validly advertised.
