# ContentBlock record

The discriminated union of content blocks carried by tool results and prompt messages (spec §14.4). The wire discriminator is the case-sensitive `type` field (R-14.4-a).

## Remarks

Five member types are recognised: [`TextContent`](./TextContent.md), [`ImageContent`](./ImageContent.md), [`AudioContent`](./AudioContent.md), [`ResourceLink`](./ResourceLink.md), and [`EmbeddedResource`](./EmbeddedResource.md).

An unknown`type` is forward-compatible: it deserialises to [`UnsupportedContentBlock`](./UnsupportedContentBlock.md) rather than failing the enclosing message (R-14.4-b). The sole exception is the deprecated sampling discriminators `tool_use` and `tool_result`, which MUST be rejected even on the fallback path (R-14.8-a, R-14.8-b). This dispatch is performed by [`ContentBlockConverter`](./ContentBlockConverter.md) so the unknown-type tolerance and the forbidden-type rejection are both honoured (STJ's built-in polymorphism would instead throw on any unknown discriminator).
