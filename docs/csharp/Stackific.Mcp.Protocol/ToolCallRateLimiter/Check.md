# ToolCallRateLimiter.Check method

Records and evaluates one `tools/call` for *key*. Returns `Allowed = true` when within the limit, or `Allowed = false` with a back-off hint when it exceeds it and MUST be rejected rather than executed (R-28.3-h). A rejected call is NOT counted toward the window, so a flood cannot extend the back-off indefinitely.
