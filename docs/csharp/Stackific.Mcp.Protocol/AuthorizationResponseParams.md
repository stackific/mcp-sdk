# AuthorizationResponseParams record

The redirect query parameters the authorization server returns (spec §23.5, §23.7). On success [`Code`](./AuthorizationResponseParams/Code.md) is present; [`State`](./AuthorizationResponseParams/State.md) echoes the request `state`; [`Iss`](./AuthorizationResponseParams/Iss.md) identifies the authorization server (SHOULD). On error, the [`Error`](./AuthorizationResponseParams/Error.md) fields are present and MUST NOT be acted on when `iss` validation fails (R-23.7-h).
