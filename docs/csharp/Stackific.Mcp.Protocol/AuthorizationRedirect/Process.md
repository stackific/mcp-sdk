# AuthorizationRedirect.Process method

Processes a Step-3 authorization redirect end to end: parses the response, verifies `state`, validates `iss` per §23.7, and only then yields the code for redemption (spec §23.5 Step 3, R-23.5-h, R-23.5-l, R-23.5-m, R-23.7-a, R-23.7-h).

```csharp
public static AuthorizationRedirectResult Process(string redirect, string recordedIssuer, 
    string? sentState = null, bool? issParameterSupported = null)
```

| parameter | description |
| --- | --- |
| redirect | The raw redirect URL or query string. |
| recordedIssuer | The Step-1 recorded issuer. |
| sentState | The Step-1 sent `state`, if any. |
| issParameterSupported | The AS metadata flag, if advertised. |

## Return Value

The redirect-processing result.

## Remarks

On an error response, the `error`/`error_description`/`error_uri` are surfaced ONLY when `iss` validation succeeds; on `iss` mismatch they are withheld and MUST NOT be acted on or displayed (R-23.7-h).
