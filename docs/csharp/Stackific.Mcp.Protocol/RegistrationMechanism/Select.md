# RegistrationMechanism.Select method

Selects the `client_id` mechanism from the validated authorization-server metadata and the client's credential state, applying the §23.11 priority order and the metadata gates (R-23.11-a – R-23.11-e).

```csharp
public static RegistrationMechanismSelection Select(AuthorizationServerMetadata metadata, 
    bool hasPreRegisteredCredentials = false, 
    IEnumerable<ClientIdMechanism>? supportedMechanisms = null)
```

| parameter | description |
| --- | --- |
| metadata | The validated authorization-server metadata (its CIMD flag and `registration_endpoint` gate the mechanisms). |
| hasPreRegisteredCredentials | `true` when the client already holds pre-registered credentials for this AS (R-23.11-b). |
| supportedMechanisms | The mechanisms this client is capable of; defaults to pre-registration, CIMD, and DCR (the prompt is always the final fallback). |

## Return Value

The selected mechanism and the reason it applied.

## Remarks

The order, using the first that applies: (1) pre-registration when credentials are already held; (2) CIMD only when the metadata sets `client_id_metadata_document_supported: true` (R-23.11-d) AND the client supports it; (3) DCR only when the metadata advertises a `registration_endpoint` (R-23.11-e) AND the client supports it; (4) otherwise prompt the user. The metadata is inspected before deciding (R-23.11-c), and CIMD/DCR are never returned when their gate is closed.
