# Resources.GuardNonEmptyContents method

Asserts a `resources/read` result's `contents` array is non-empty, throwing `-32603` (Internal error) otherwise. A server MUST NOT use an empty `contents` array to signal non-existence — that case is the `-32602` not-found error ([`BuildResourceNotFoundError`](./BuildResourceNotFoundError.md)). When a handler returns empty contents for a URI it claims to serve, that is a server-side fault, hence the internal-error code. (§17.5, R-17.5-z, R-17.5-aa)
