# MultiRoundTrip.ComputeRetryBackoffMs method

Computes an exponential-backoff delay (ms) for the *attempt*-th retry on repeated non-progress (§11.5, R-11.5-n). A client retrying without progress SHOULD apply a reasonable backoff.

```csharp
public static long ComputeRetryBackoffMs(int attempt, long baseMs = 250, long maxMs = 30000)
```

| parameter | description |
| --- | --- |
| attempt | The 1-based retry attempt number (≤ 0 ⇒ 0 ms). |
| baseMs | The base delay (default 250). |
| maxMs | The cap (default 30000). |

## Return Value

The backoff delay in milliseconds.
