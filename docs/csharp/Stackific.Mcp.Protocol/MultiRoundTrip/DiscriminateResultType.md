# MultiRoundTrip.DiscriminateResultType method

Branches on the `resultType` of a received result per the normative client-side rules of §11.5 (R-11.5-c, R-11.5-d, R-11.5-e, R-11.5-f, R-11.5-k):

* `"complete"` or absent/null `resultType` → Complete.
* `"input_required"` with a valid result → InputRequired.
* Any unrecognized `resultType`, a malformed `input_required` result, or (when *clientCapabilities* is supplied) an UNDECLARED requested input-request kind → Error (R-11.5-d, R-11.5-k).

```csharp
public static ResultDiscrimination DiscriminateResultType(JsonObject? result, 
    JsonObject? clientCapabilities = null)
```

| parameter | description |
| --- | --- |
| result | The raw result object received from the wire (`null` ⇒ error). |
| clientCapabilities | The client's declared capabilities; when supplied, every requested input-request kind is gated against them and an undeclared kind makes the whole result an error (R-11.5-k). When `null`, the discrimination is capability-blind (back-compat). |

The discrimination outcome.
