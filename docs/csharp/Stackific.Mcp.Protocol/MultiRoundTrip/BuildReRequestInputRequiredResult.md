# MultiRoundTrip.BuildReRequestInputRequiredResult method

Builds a NEW [`InputRequiredResult`](../InputRequiredResult.md) re-requesting only the still-missing input, or `null` when the retry supplied everything (§11.5, R-11.5-q). A server whose retry `inputResponses` is well-formed but incomplete SHOULD re-request the missing information rather than failing the request.
