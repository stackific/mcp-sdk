# RootsValidation.ApplyNonFileDisposition method

Applies a receiver's chosen *disposition* to a candidate root *uri* that does NOT use the `file` scheme, returning whether the root is kept. A valid `file://` URI is always kept; a non-`file` URI is dropped under EITHER disposition (they differ only in whether the receiver surfaces an error elsewhere). Mirrors TS `applyNonFileDisposition`. (R-21.1.5-c; AC-32.12)
