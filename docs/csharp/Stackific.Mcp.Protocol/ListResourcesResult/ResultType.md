# ListResourcesResult.ResultType property

REQUIRED on emit (§3.6, §17.2): the base result-type discriminator, `"complete"` for a list page. Defaults to `"complete"` so a constructed result is well-formed, and — because it is plain `init`-settable — an inbound result that OMITS `resultType` deserializes to this default, which is exactly the §3.6 receiver rule ("treat an absent `resultType` as if it were `complete`").
