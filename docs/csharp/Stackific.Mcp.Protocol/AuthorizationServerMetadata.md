## Remarks

[`Issuer`](./AuthorizationServerMetadata/Issuer.md), [`AuthorizationEndpoint`](./AuthorizationServerMetadata/AuthorizationEndpoint.md), and [`TokenEndpoint`](./AuthorizationServerMetadata/TokenEndpoint.md) are REQUIRED (R-23.3-f – R-23.3-h). When present, [`ResponseTypesSupported`](./AuthorizationServerMetadata/ResponseTypesSupported.md) MUST include `code` (R-23.3-i) and [`CodeChallengeMethodsSupported`](./AuthorizationServerMetadata/CodeChallengeMethodsSupported.md) (OPTIONAL but RECOMMENDED) MUST include `S256` (R-23.3-j) — both enforced by [`Parse`](./AuthorizationServerMetadata/Parse.md). The issuer-match check (R-23.3-d, R-23.3-e) is applied at validation time (it depends on the fetch URL).
