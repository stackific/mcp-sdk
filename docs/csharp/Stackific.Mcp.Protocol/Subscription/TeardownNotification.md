# Subscription.TeardownNotification method

Builds the server-teardown signal for this subscription: a `notifications/cancelled` referencing the `subscriptions/listen` request id (§10.7, R-10.7-b). On stdio the transport sends this after `Close(ServerTeardown)`; on Streamable HTTP it instead ends the SSE response. The `params.requestId` equals this subscription's listen id so the client can correlate.
