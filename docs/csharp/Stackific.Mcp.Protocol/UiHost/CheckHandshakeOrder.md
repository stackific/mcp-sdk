# UiHost.CheckHandshakeOrder method

Conformance check for the handshake-ordering rule (spec §26.5.1, R-26.5.1-a): given the channel *phase* and the *method* the UI is attempting to send, returns `Ok` when the message is allowed, or a premature-message violation when the UI emits anything other than `ui/initialize` before the init response.

public static HandshakeOrderResult CheckHandshakeOrder(ChannelPhase phase, string method)

| parameter | description |

| method | The method/notification name the UI is attempting to send. |
