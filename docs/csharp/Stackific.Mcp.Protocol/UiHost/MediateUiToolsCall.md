# UiHost.MediateUiToolsCall method

Decides whether a host may route a UI-initiated `tools/call` to the server (spec §26.5.3, §26.7, R-26.5.3-a, R-26.5.3-b, R-26.7-i, R-26.7-j, R-26.7-k).

```csharp
public static MediationDecision MediateUiToolsCall(ToolsCallMediationInput input)
```

| parameter | description |
| --- | --- |
| input | The host's mediation inputs. |

## Return Value

The mediation decision.

## Remarks

The host routes the call ONLY when ALL hold, in this precedence:

1. the tool's effective `visibility` includes `"app"` (SHOULD reject otherwise — reuses [`HostShouldRejectUiOriginatedCall`](../Ui/HostShouldRejectUiOriginatedCall.md)); a rejection here is a Policy decline;
2. the host's tool-execution policy permits the call (Policy);
3. the user has consented (NoConsent).

A path that reaches the server WITHOUT prior consent and policy is a failure: this function returns `Route == false` in every such case, and the caller MUST answer with the corresponding §22 error (never a silent drop).
