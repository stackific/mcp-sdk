# ContinuationTokenStore&lt;TState&gt; class

A server-side store for `requestState` continuation tokens that protects their integrity and confidentiality and guards against replay — the §28.6 handling profile (spec §28.6, R-28.6-a, R-28.6-b, R-28.6-c; AC-44.18).

## Remarks

The client only ever sees the opaque `value`; the state and integrity tag are held entirely server-side (the "unguessable handle" design §28.6 permits). On presentation [`Validate`](./ContinuationTokenStore-1/Validate.md) rejects — rather than acting on — a token that fails integrity (R-28.6-b), is expired, was already consumed (single-use replay defense), or is unknown. [`Issue`](./ContinuationTokenStore-1/Issue.md) mints a single-use, optionally time-bounded handle.
