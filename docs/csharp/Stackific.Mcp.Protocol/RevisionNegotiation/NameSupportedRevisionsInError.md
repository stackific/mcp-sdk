# RevisionNegotiation.NameSupportedRevisionsInError method

Adds the server's supported revisions to an error's `data.supported` so a peer with no fall-forward mechanism can still surface a useful diagnostic (§5.7, R-5.7-g). Existing `data` fields are preserved; `supported` is set/overwritten. Never mutates the input. Mirrors TS `nameSupportedRevisionsInError`.
