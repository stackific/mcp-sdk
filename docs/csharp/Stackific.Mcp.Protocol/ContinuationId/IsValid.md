# ContinuationId.IsValid method

Returns `true` when *value* is a JSON-serializable value that may serve as a continuation identifier (spec §4.5 / R-4.5-b). A continuation id must round-trip through JSON without loss.

```csharp
public static bool IsValid(object? value)
```

| parameter | description |
| --- | --- |
| value | The candidate continuation identifier. |

## Return Value

`true` when the value can serve as an opaque continuation id.

## Remarks

In the TypeScript SDK the excluded forms are `undefined`, `function`, `symbol`, and `bigint`. In the .NET model the JSON value space is represented by JsonNode (objects, arrays, and JsonValue scalars) and the .NET `null` literal — which stands for the JSON `null` a server is permitted to mint. A non-JSON CLR object (anything that is neither a JsonNode nor a primitive the JSON value model admits) is rejected, mirroring the TypeScript exclusions.
