# AuthorizationSecurity.RefreshTokenAlwaysOptional field

`true` — a client MUST NOT assume a refresh token will be issued; the authorization server retains discretion (R-23.19-t). This is an unconditional control-flow invariant (treat the refresh token as optional and handle its absence, pairing with [`HasNoRefreshToken`](../TokenResponse/HasNoRefreshToken.md)), so it is a named constant rather than a no-argument predicate.
