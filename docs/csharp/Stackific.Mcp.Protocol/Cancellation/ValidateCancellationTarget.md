# Cancellation.ValidateCancellationTarget method

Validates that a cancellation target (the `requestId` from a `notifications/cancelled`) is eligible given the sender's in-flight set. A valid target MUST be present, appear in *inFlightIds* (in-flight from the sender's perspective), and not be the `server/discover` id when *discoverRequestId* is supplied. Mirrors TS `validateCancellationTarget`. (R-15.2.1-a/b, R-15.2.2-b)

```csharp
public static CancellationValidationResult ValidateCancellationTarget(RequestId? requestId, 
    IReadOnlySet<RequestId> inFlightIds, RequestId? discoverRequestId = default)
```

| parameter | description |
| --- | --- |
| requestId | The cancellation target, or `null` when the notification omitted it. |
| inFlightIds | The ids the sender has issued and not yet received a response to. |
| discoverRequestId | The `server/discover` id that must not be cancelled, or `null`. |

## Return Value

The validation result; on failure carrying a reason.
