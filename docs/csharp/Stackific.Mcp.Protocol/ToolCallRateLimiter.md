# ToolCallRateLimiter class

A sliding-window rate limiter a server applies to `tools/call` so a hostile or malfunctioning client cannot drive unbounded execution or downstream load (spec §28.3, R-28.3-g, R-28.3-h; AC-44.9).

## Remarks

[`Check`](./ToolCallRateLimiter/Check.md) returns whether a call is within the limit; a server MUST reject (not execute) any call that exceeds it (R-28.3-h) — use [`BuildRateLimitRejection`](./Security/BuildRateLimitRejection.md) to build the `-32600` error. The window is keyed by an opaque caller-chosen client/session id so per-peer limits are independent. Time is injectable for testing.
