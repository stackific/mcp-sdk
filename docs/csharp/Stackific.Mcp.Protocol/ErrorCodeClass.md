# ErrorCodeClass enumeration

The three (plus extension) classes a JSON-RPC error code can fall into, per §22. The numeric code is authoritative; this taxonomy lets a receiver reason about a code it has never seen (R-22.1-h, R-22.7-a, R-22.7-e).

## Values

| name | value | description |
| --- | --- | --- |
| JsonRpcStandard | `0` | The reserved JSON-RPC pre-defined codes (`-32700`, `-32600..-32603`). |
| McpProtocol | `1` | MCP protocol-specific codes (`-32003`, `-32004`, `-32002`) with normative data (§22.3). |
| ServerDefined | `2` | The implementation-defined server-error range `-32000..-32099` (§22.7). |
| ExtensionDefined | `3` | Any integer outside every reserved/server range — extension-defined (§22.7). |
