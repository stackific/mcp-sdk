# ToolSchemas.ValidateToolSchema method

Validates a tool's `inputSchema` or `outputSchema` against the §16.4 hardening rules, WITHOUT any network or file-system retrieval. Returns a structured result rather than throwing so a caller can reject-or-refuse-registration. (§16.4, R-16.4-d/e/f/g/k/l/n/s/t)

```csharp
public static ToolSchemaValidation ValidateToolSchema(JsonNode? schema, ToolSchemaRole role, 
    SchemaLimits? limits = default, bool allowExternalRefs = false)
```

| parameter | description |
| --- | --- |
| schema | The raw schema node. |
| role | Whether the schema is an input (root must be `object`) or output schema. |
| limits | Resource bounds; defaults to [`Default`](../SchemaLimits/Default.md). |
| allowExternalRefs | Opt-in non-local `$ref` fetching; defaults to `false`. |

## Return Value

A [`ToolSchemaValidation`](../ToolSchemaValidation.md) describing acceptance or rejection.

## Remarks

Checks, in order: (1) the schema is a JSON object — not `null`, not an array, not a scalar (R-16.4-n); (2) its declared/default dialect is supported (R-16.4-t/s); (3) nesting depth and node count are within *limits* (R-16.4-l/m/n); (4) unless *allowExternalRefs* is `true` (default `false`, R-16.4-i), no external `$ref`/`$dynamicRef` is present (R-16.4-f/g/k); (5) for an input schema, the root `type` MUST be `"object"` (R-16.4-d); an output schema's root type is unrestricted (R-16.4-e).
