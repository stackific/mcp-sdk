# InputRequiredResult record

The `input_required` result a server returns to solicit client input while processing a request (spec §11.2). At least one of [`InputRequests`](./InputRequiredResult/InputRequests.md) or [`RequestState`](./InputRequiredResult/RequestState.md) MUST be present; a result with only [`RequestState`](./InputRequiredResult/RequestState.md) is the load-shedding (retry-later) signal (§11.5). The base `resultType` is supplied by the runtime as `input_required`.
