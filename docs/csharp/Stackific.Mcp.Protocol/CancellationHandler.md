# CancellationHandler class

A receiver-side registry mapping in-flight request ids to abort callbacks — the C# counterpart of the TypeScript `CancellationHandler`. When a valid `notifications/cancelled` arrives, the receiver SHOULD stop processing the matching request, free resources, and suppress the response; this type wires that behavior (R-15.2.2-d / RC-4).

## Remarks

Lifecycle: [`Register`](./CancellationHandler/Register.md) before dispatching a long-running request; [`Trigger`](./CancellationHandler/Trigger.md) when a valid cancellation arrives (fires the callback and removes the entry); [`Deregister`](./CancellationHandler/Deregister.md) on normal completion (removes the entry WITHOUT firing). String and number ids are tracked independently (the [`RequestId`](../Stackific.Mcp.JsonRpc/RequestId.md) identity preserves the JSON type). Not thread-safe.
