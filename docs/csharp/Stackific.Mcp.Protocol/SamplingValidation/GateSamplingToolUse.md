# SamplingValidation.GateSamplingToolUse method

Client-side gate: returns a failed result when a tool-enabled sampling request arrives but the client did not declare `sampling.tools` (R-21.2.3-b, R-21.2.4-n/o). `tools` is checked before `toolChoice` so the error names the first offending field deterministically. When `sampling.tools` is declared, or the request is not tool-enabled, the gate passes. Mirrors TS `gateSamplingToolUse`.
