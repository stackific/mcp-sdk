## Remarks

A client MUST store this separately per authorization server (R-23.1-i); credentials registered with one server MUST NOT be assumed valid at another (R-23.1-j). The concrete [`ClientId`](./AuthorizationServerRegistration/ClientId.md)/token fields are populated by S36/S37 — this story owns only the per-`issuer` isolation contract.
