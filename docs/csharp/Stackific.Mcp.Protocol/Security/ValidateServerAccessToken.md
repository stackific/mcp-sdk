# Security.ValidateServerAccessToken method

Validates, server-side, that a presented access token is audience-bound to THIS server and was validated before the request is processed; rejects otherwise so no data is returned to an unauthorized party (spec §28.5, R-28.5-b … R-28.5-e; AC-44.12). The audience claim may be a single resource (string) or an array; it matches when it includes the canonical resource.

```csharp
public static ServerTokenValidation ValidateServerAccessToken(IReadOnlyList<string> tokenAudience, 
    string ownCanonicalResource, bool validatedBeforeUse)
```

| parameter | description |
| --- | --- |
| tokenAudience | The `aud` claim the presented token carries (string or array of strings). |
| ownCanonicalResource | This server's canonical resource identifier. |
| validatedBeforeUse | `true` when the token was cryptographically validated before processing (R-28.5-d). |
