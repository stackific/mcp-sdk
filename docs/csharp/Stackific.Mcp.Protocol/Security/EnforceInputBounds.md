# Security.EnforceInputBounds method

Bounds the resources consumed while validating a peer input: rejects a schema whose nesting depth exceeds the limit and a payload exceeding the size limit (spec §28.10, R-28.10-k, R-28.10-l; AC-44.28). The depth probe stops at the cap so a pathological self-referential schema cannot exhaust the stack while being measured. The payload-size check uses the UTF-8 byte length.

```csharp
public static InputBoundsValidation EnforceInputBounds(JsonNode? schema = null, 
    string? serializedPayload = null, InputBounds? bounds = default)
```

| parameter | description |
| --- | --- |
| schema | The schema to depth-bound, or `null` (R-28.10-k). |
| serializedPayload | An optional serialized payload whose size is bounded (R-28.10-l). |
| bounds | The bounds to enforce; defaults to [`DefaultInputBounds`](./DefaultInputBounds.md). |
