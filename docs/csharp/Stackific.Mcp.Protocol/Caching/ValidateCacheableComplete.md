# Caching.ValidateCacheableComplete method

Asserts the emit-side invariant for a cacheable result that is being sent with a completed (`"complete"`) discriminator (§3.6, §13.1, §13.4): the *resultType* MUST be exactly `"complete"`, and BOTH caching hints MUST be present — *ttlMs* a non-negative integer and *cacheScope* a resolved scope. A sender MUST NOT emit one hint without the other (§13.1), MUST NOT emit a negative `ttlMs` (§13.2), and MUST carry the discriminator (§3.6). This is the construction-time guard a server applies before a result reaches the wire; it is the counterpart of the lenient receive-side degradation in [`IsCacheHintValid`](./IsCacheHintValid.md)/[`ResolveCacheScope`](./ResolveCacheScope.md), which never throws.

| parameter | description |
| --- | --- |
| resultType | The result's discriminator (§3.6). |
| ttlMs | The freshness hint in milliseconds, or `null` when absent. |
| cacheScope | The sharing scope, or `null` when absent. |
| resultName | The result type name, used in the exception message. |
