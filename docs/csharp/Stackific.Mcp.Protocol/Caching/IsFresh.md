# Caching.IsFresh method

Returns `true` when the result is still within its freshness window: `(ttlMs > 0) AND (now < receivedAt + ttlMs)`. A `ttlMs` of `0` is never fresh (immediately stale). The computation uses ONLY the client's local *receivedAt* and the *ttlMs* value — it MUST NOT assume the client and server clocks agree. Mirrors TS `isFresh`. (R-13.2-e/f/g)

```csharp
public static bool IsFresh(long ttlMs, long receivedAt, long now)
```

| parameter | description |
| --- | --- |
| ttlMs | The non-negative freshness hint in milliseconds. |
| receivedAt | The client-local timestamp (ms) when the response was received. |
| now | The current client-local timestamp (ms). |

## Return Value

`true` when the result is still fresh.
