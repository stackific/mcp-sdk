# Tasks.IsTaskBackstopElapsed method

Returns `true` when a client MAY treat a task as no longer usable because its non-null *ttlMs* backstop has elapsed without the observable status advancing past a non-terminal state (§25.11, R-25.11-c). A `null`*ttlMs* is never a backstop, and an already-terminal task is never backstopped.

```csharp
public static bool IsTaskBackstopElapsed(long createdAtMs, long? ttlMs, long nowMs, 
    McpTaskStatus status)
```

| parameter | description |
| --- | --- |
| createdAtMs | The task's creation time in epoch milliseconds. |
| ttlMs | The task's `ttlMs` (a non-negative number, or `null`). |
| nowMs | The current time in epoch milliseconds. |
| status | The task's last observed status. |

## Return Value

`true` when the backstop has elapsed and the task is still non-terminal.
