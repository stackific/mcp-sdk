# Tasks.HasConsistentInlineOutcome method

Returns `true` when a [`DetailedTask`](../DetailedTask.md) observes the §25.5 inline-outcome rule (R-25.5-d): a non-terminal or `cancelled` task carries neither `result` nor `error`; a `completed` task carries `result` (and no `error`); a `failed` task carries `error` (and no `result`).
