# AuthorizationServerRegistration constructor

Registration state held for a single authorization server, keyed by its `issuer` (spec §23.1).

```csharp
public AuthorizationServerRegistration(string Issuer, string? ClientId = null, 
    string? AccessToken = null, string? RefreshToken = null)
```

| parameter | description |
| --- | --- |
| Issuer | The authorization server's `issuer` identifier URL; the isolation key. |
| ClientId | OPTIONAL registered client identifier (populated by S36/S37). |
| AccessToken | OPTIONAL issued access token (populated by S36). |
| RefreshToken | OPTIONAL issued refresh token (populated by S36). |

## Remarks

A client MUST store this separately per authorization server (R-23.1-i); credentials registered with one server MUST NOT be assumed valid at another (R-23.1-j). The concrete [`ClientId`](./ClientId.md)/token fields are populated by S36/S37 — this story owns only the per-`issuer` isolation contract.
