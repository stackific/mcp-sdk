# RegistrationMechanism class

Metadata-driven `client_id` mechanism selection: choosing the first applicable mechanism after inspecting the validated authorization-server metadata, gating CIMD on `client_id_metadata_document_supported` and DCR on `registration_endpoint` (spec §23.11, R-23.11-a – R-23.11-e).

## Remarks

This complements S36's [`Select`](./ClientIdAcquisition/Select.md), which ranks a static capability set; here the live metadata flags and the pre-registration credential state are the deciding inputs.
