# ErrorRegistry.ClassifyToolCallFailure method

Decides whether a `tools/call` failure is reported as a JSON-RPC protocol error (`-32602`) or as a successful result with `isError: true` (R-22.5-a..f, AC-34.18).

```csharp
public static ToolFailureMechanism ClassifyToolCallFailure(ToolCallFailureSituation situation)
```

| parameter | description |
| --- | --- |
| situation | The situation the failure arose from. |

## Return Value

The mechanism the failure MUST be reported with.

## Remarks

Undispatchable / schema-invalid requests (UnknownTool, InvalidArguments) are protocol errors and MUST never produce `isError: true`; a tool that ran and failed (ExecutionFailure) is an error result and MUST never produce a JSON-RPC error. The mapping is total and never the reverse.
