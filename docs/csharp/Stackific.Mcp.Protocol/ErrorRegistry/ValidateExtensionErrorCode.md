# ErrorRegistry.ValidateExtensionErrorCode method (2 of 2)

Validates that *code* is a legal extension-defined error code: an integer that does not collide with any reserved code (R-22.7-a..c).

```csharp
public static ExtensionCodeValidation ValidateExtensionErrorCode(int code)
```

| parameter | description |
| --- | --- |
| code | The candidate extension error code. |

## Return Value

An [`ExtensionCodeValidation`](../ExtensionCodeValidation.md) describing whether the code is usable.

## Remarks

Because the parameter is already a CLR Int32, the TypeScript `not-an-integer` outcome (which guards a JavaScript `number` that might be fractional) is unreachable from strongly-typed C# callers; the reason is retained on [`ExtensionCodeValidation`](../ExtensionCodeValidation.md) for parity with the TS contract and is returned when a caller deserialises a non-integer code into this check via the [`ValidateExtensionErrorCode`](./ValidateExtensionErrorCode.md) overload.
