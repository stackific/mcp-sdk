# ProgressTracker class

Tracks active progress tokens for a single sender, enforcing the §15.1 uniqueness and monotonicity rules — the C# counterpart of the TypeScript `ProgressTracker`. A token's identity preserves the JSON type, so string `"1"` and number `1` are distinct (R-15.1.1-c).

## Remarks

Rules enforced: tokens MUST be unique across the sender's active requests (R-15.1.1-c); receivers MUST treat the token as opaque (R-15.1.1-d); `progress` MUST strictly increase across successive notifications (R-15.1.3-e); progress MUST stop once the operation reaches a terminal state (R-15.1.4-g — modeled by [`Complete`](./ProgressTracker/Complete.md) removing the token). Not thread-safe.
