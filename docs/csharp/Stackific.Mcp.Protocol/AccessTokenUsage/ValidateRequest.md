# AccessTokenUsage.ValidateRequest method

Validates a presented access token on the MCP server side, on EVERY request, yielding a `401`/`403` challenge on failure (R-23.8-a, R-23.8-d, R-23.8-e, R-23.8-f).

```csharp
public static AccessTokenValidationResult ValidateRequest(PresentedToken? token, 
    string ownCanonicalResource, string resourceMetadata, 
    IReadOnlyList<string>? requiredScopes = null)
```

| parameter | description |
| --- | --- |
| token | The presented token's validated facts, or `null` when absent. |
| ownCanonicalResource | This server's canonical resource identifier (the expected audience). |
| resourceMetadata | The protected-resource metadata URI for the challenge. |
| requiredScopes | The scopes this operation requires; empty/`null` when none. |

## Return Value

The authorization outcome (authorized, or a challenge).

## Remarks

The checks, in order: missing / inactive / expired token → `401` (R-23.8-e); wrong audience → `401` (R-23.6-f/g, R-23.8-d/e); valid token lacking a required scope → `403` with an `insufficient_scope` challenge (R-23.8-f).
