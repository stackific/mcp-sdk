# McpTask record

The handle and status record for a long-running operation (spec §25.4). A [`McpTask`](./McpTask.md) is returned immediately in place of a blocking result; the client polls `tasks/get` to observe progress until the task reaches a terminal status. Identifiers are server-minted and opaque, and the durable record survives client disconnects and restarts (§25.6). Named `McpTask` to avoid clashing with Task; the wire object is unaffected.
