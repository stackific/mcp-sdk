# TaskStatusNotificationParams record

The parameters of the `notifications/tasks` notification (spec §25.10), by which a server pushes a task state change. The params are a complete [`DetailedTask`](./DetailedTask.md) for the task's current status — identical to what `tasks/get` would have returned at that moment — so a subscribed client need not issue an extra `tasks/get`. Delivery is opt-in via a `taskIds` subscription filter on `subscriptions/listen` (§10, §25.10).
