# ProgressToken structure

A progress-correlation token (spec §15.1.1): a value that is either a JSON string or a JSON number, never `null`. The token is carried in an originating request's `_meta.progressToken` (§15.1.2) and is echoed on every [`ProgressNotificationParams`](./ProgressNotificationParams.md) so that out-of-band progress can be matched to the request that opted in.

## Remarks

The wire type is preserved exactly — a numeric token round-trips as a number and a string token as a string, and the two are never coerced into one another. Receivers MUST treat the token as opaque (§15.1.1).
