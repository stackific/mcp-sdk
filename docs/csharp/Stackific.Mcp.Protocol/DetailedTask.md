# DetailedTask record

A [`McpTask`](./McpTask.md) that additionally conveys the terminal payload (or pending input requests) inline (spec §25.4). It is the shape returned by `tasks/get` and carried on `notifications/tasks`, modelled here as a single record whose status-specific members are populated according to `status`: [`InputRequests`](./DetailedTask/InputRequests.md) for `input_required`, [`Result`](./DetailedTask/Result.md) for `completed`, and [`Error`](./DetailedTask/Error.md) for `failed`. A non-terminal task carries neither [`Result`](./DetailedTask/Result.md) nor [`Error`](./DetailedTask/Error.md) (§25.5).
