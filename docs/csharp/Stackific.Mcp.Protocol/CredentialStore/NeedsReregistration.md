# CredentialStore.NeedsReregistration method

Returns `true` when moving from *previousIssuer* to *currentIssuer* requires the client to re-register / re-discover rather than reuse credentials: whenever the indicated authorization server changed, or no credentials are yet stored for *currentIssuer*. A client MUST NOT reuse a different server's credentials (R-23.1-k) and MUST re-register/re-discover against the new one (R-23.1-l).

```csharp
public bool NeedsReregistration(string? previousIssuer, string currentIssuer)
```

| parameter | description |
| --- | --- |
| previousIssuer | The previously indicated `issuer`, or `null` when none was. |
| currentIssuer | The `issuer` now indicated by protected-resource metadata. |

## Return Value

`true` when re-registration/re-discovery is required.
