# ConformanceRequirements.ValidateInputRequiredRetry method

Validates a client's retry request after an `input_required` result (spec §29.3 item 4, R-29.3-j). The retry MUST use a request id distinct from the original, echo `requestState` byte-for-byte when one was provided, and omit it when none was provided. Identity comparison uses [`RequestId`](../../Stackific.Mcp.JsonRpc/RequestId.md) value equality, which preserves the JSON type (a numeric and a string id are never coerced); `requestState` comparison is ordinal string equality (the value is opaque and echoed exactly, R-29.3-f).

```csharp
public static InputRequiredRetryValidation ValidateInputRequiredRetry(RequestId originalId,

```

| parameter | description |

| originalId | The original request's id (a JSON string or number). |
| retryId | The retry request's id (must differ from the original). |
| providedState | The `requestState` the server provided, or `null` when none. |
| retryState | The `requestState` the retry carries, or `null` when absent. |

The validation outcome.
