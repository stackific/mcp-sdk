# Extensions.IsIncompatibleChange method

Returns `true` when a change of *kind* is INCOMPATIBLE — it would cause an existing implementation to fail or behave incorrectly — and therefore SHOULD be published under a new extension identifier (spec R-24.6-d). Backward-compatible changes return `false`.
