# Subscription class

Tracks the request-scoped lifecycle of a single client-side subscription (spec §10.7). The state is scoped to the `subscriptions/listen` request, NOT to the connection: once closed the subscription is gone and retains NO resumable state — re-establishment is a NEW `subscriptions/listen` request yielding a NEW id (R-10.7-d, R-10.7-f). The C# counterpart of the TypeScript `Subscription` class.
