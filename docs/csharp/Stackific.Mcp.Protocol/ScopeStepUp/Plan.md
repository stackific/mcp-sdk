# ScopeStepUp.Plan method

Plans one step-up re-authorization end to end: decides whether to proceed (by actor and remaining retries), computes the UNION scope set that never drops already-granted scopes, and records the attempt against the bound (R-23.18-l – R-23.18-r, R-23.1-ae – R-23.1-ag).

```csharp
public static StepUpPlan Plan(StepUpActor actor, IReadOnlyList<string> alreadyGranted, 
    WwwAuthenticateChallenge challenge, ScopeUpgradeKey key, ScopeUpgradeTracker tracker, 
    bool forceForClientCredentials = false)
```

| parameter | description |
| --- | --- |
| actor | Who the client is acting for (R-23.18-m, R-23.18-n). |
| alreadyGranted | The scopes the client already holds/requested (R-23.18-o). |
| challenge | The challenge driving the step-up (its `scope` is parsed for the union) (R-23.18-l). |
| key | The resource-and-operation being upgraded, for retry tracking (R-23.18-r). |
| tracker | The shared upgrade tracker enforcing the retry bound (R-23.18-q). |
| forceForClientCredentials | `true` to attempt step-up even for a ClientCredentials client, exercising the MAY of R-23.18-n. Defaults to `false`. |

## Return Value

The step-up plan.

## Remarks

Proceeds when (a) the actor SHOULD/elects to step up — a user-acting client, or a ClientCredentials client with *forceForClientCredentials* — AND (b) the tracker still permits a retry for the *key*. When it proceeds it records the attempt (R-23.1-ag) and returns the unioned scopes for a fresh authorization-code+PKCE flow. When the retry bound is exhausted it returns `Proceed: false` so the caller treats the failure as permanent (R-23.18-q).
