# ReadResourceResult.ResultType property

REQUIRED on emit (§3.6, §17.5): the base result-type discriminator, `"complete"` for a completed read. Defaults to `"complete"`, which also makes an inbound result that omits `resultType` degrade to `complete` per the §3.6 receiver rule (R-17.5-q). A read reply MAY instead carry `"input_required"` (§11); discriminate with [`DiscriminateReadResourceResponse`](../Resources/DiscriminateReadResourceResponse.md) before binding to this record.
