## Remarks

A consumer MUST accept only `https:` URLs or `data:` URIs (R-14.2-o, AC-20.22) and MUST reject `javascript:`, `file:`, `ftp:`, `ws:`, and other unsafe schemes (R-14.2-n, AC-20.21). Note that `http:` is also rejected: R-14.2-o's stricter consumer rule supersedes the field description in R-14.2-d.
