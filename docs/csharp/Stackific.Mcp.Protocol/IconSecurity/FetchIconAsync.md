# IconSecurity.FetchIconAsync method

Securely fetches and validates an icon, enforcing the §14.2 transport rules.

## Remarks

Security rules enforced:

* *src* MUST be `https:` or `data:` (R-14.2-o, via [`ValidateIconSrc`](./ValidateIconSrc.md)).
* Redirects are followed manually; a redirect that changes the scheme or moves to a different origin is rejected (R-14.2-p, TV-20.12).
* The request is credential-free: no cookies and no `Authorization` header are ever sent (R-14.2-q, TV-20.13).
* The returned bytes are validated against the allowlist by magic bytes, ignoring the declared type (R-14.2-r – R-14.2-u, via [`ValidateIconBytes`](./ValidateIconBytes.md)).

Tests inject a custom HttpMessageHandler (or a pre-built HttpClient) so they never hit the network. A `data:` source carries its bytes inline and performs no request at all.
