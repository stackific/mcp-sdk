# Dcr.HandleResponse method

Handles a DCR registration response, surfacing a meaningful error on failure and flagging whether a retry may help, rather than crashing (R-23.4-p, R-23.4-q, R-23.4-r). A success body (valid JSON with a `client_id`) yields success; an HTTP failure status, or a body lacking `client_id`, yields a failure with a human-readable reason. `Retryable` is `true` for a `400` (typically a redirect-URI / application-type constraint).
