# RequestMeta.IsValidRevisionFormat method

Returns `true` when *revision* matches the `YYYY-MM-DD` revision-format (§5.1, R-5.2-b). Mirrors the TypeScript `isValidRevisionFormat` / `PROTOCOL_REVISION_FORMAT_RE`.

```csharp
public static bool IsValidRevisionFormat(string revision)
```

| parameter | description |
| --- | --- |
| revision | The candidate revision identifier. |

## Return Value

`true` when the value is a well-formed `YYYY-MM-DD` string.

## Remarks

A `true` result is a shape check only — it does NOT mean the revision is supported (use [`IsSupported`](../ProtocolRevision/IsSupported.md) for that). The regex validates only the digit/separator layout, not calendar correctness, honoring the rule that revision identifiers are opaque, exactly-matched strings never compared lexically, chronologically, or by range (R-5.1-a, R-5.1-b).
