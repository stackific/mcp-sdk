# Completion.ComputeCompletion method

Caps an already-ranked candidate list at 100 and signals truncation, producing the `completion` object a server returns. Critically — matching the TS `computeCompletion` — `total` and `hasMore` are set ONLY when matches were dropped (`total > values.Count`); an under-cap result leaves them ABSENT (unknown), rather than always emitting an exact `total`. (§19.4, R-19.4-c – R-19.4-h, R-19.5-g, R-19.5-h)
