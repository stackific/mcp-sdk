# Completion.CreateCompletionDebouncer&lt;T&gt; method

Creates a client-side debouncer that coalesces rapid successive completion calls (for example one per keystroke) into a single in-flight `completion/complete` request: each call resets a *waitMs* timer, and only the final value after a quiet period is sent. All callers awaiting during a burst resolve with that single result. (§19.5, R-19.5-n — SHOULD)

```csharp
public static Func<string, Task<T>> CreateCompletionDebouncer<T>(Func<string, Task<T>> run, 
    int waitMs = 150)
```

| parameter | description |
| --- | --- |
| T | The completion result type. |
| run | Issues the actual completion request for an argument value. |
| waitMs | The quiet period (ms) before the coalesced call fires. Default 150. |

## Return Value

A debounced runner mapping a value to a task that resolves with the coalesced result.

## Remarks

The C# counterpart of the TS `createCompletionDebouncer`. Returns a delegate that maps an argument value to a Task; the returned debouncer is itself thread-safe and may be reused across bursts. Cancellation/disposal is the caller's concern (no timer leaks: the internal timer is one-shot and re-armed per call).
