# ConformanceRequirements.ServerRequestDisposition constructor

The disposition a conformant server reaches for an incoming request after the ordered §29.2 checks (spec §29.2). On a revision failure, [`Code`](./Code.md) is `-32004` and [`Supported`](./Supported.md) / [`Requested`](./Requested.md) are populated; on an envelope failure, [`Code`](./Code.md) is `-32602` with a [`Message`](./Message.md); on a capability failure, [`Code`](./Code.md) is `-32003` with [`RequiredCapabilities`](./RequiredCapabilities.md); on a gating refusal, none of these are set.

    JsonObject? RequiredCapabilities = null)

| parameter | description |
| --- | --- |

| Supported | The supported revisions on a revision failure, or `null`. |
| Requested | The requested revision on a revision failure, or `null`. |
| RequiredCapabilities | The missing capabilities on a capability failure, or `null`. |
