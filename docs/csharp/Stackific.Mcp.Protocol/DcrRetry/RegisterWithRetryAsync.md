# DcrRetry.RegisterWithRetryAsync method

Performs Dynamic Client Registration with bounded retry, surfacing a meaningful error and retrying with an adjusted `application_type` when the AS rejects on a redirect-URI / application-type constraint (R-23.15-d, R-23.15-e, R-23.15-f).

```csharp
public static Task<DcrRetryResult> RegisterWithRetryAsync(ApplicationType initialApplicationType, 
    Func<ApplicationType, Task<(int Status, JsonNode? Body)>> attempt, int maxAttempts = 2)
```

| parameter | description |
| --- | --- |
| initialApplicationType | The `application_type` for the first attempt. |
| attempt | Performs one registration POST for the given `application_type`, returning the AS's HTTP status and parsed body. Injected so this is transport-agnostic. |
| maxAttempts | The maximum number of attempts (initial plus retries); MUST be a few at most. Defaults to `2` (R-23.15-f). |

## Return Value

The final result and the attempts made.

## Remarks

Each attempt's response is interpreted by S36's [`HandleResponse`](../Dcr/HandleResponse.md); on a retryable failure the `application_type` is flipped (Native ↔ Web) for the next attempt, up to *maxAttempts*. This never throws on an AS rejection — it returns the structured failure for the client to surface (R-23.15-e).
