# ElicitRequestParams record

The parameters of an `elicitation/create` input request (spec §20.2). Elicitation lets a server request structured input from the user through the client; it is delivered as an input-required result and answered by retrying the originating request (§11). This is a closed union of two mode-specific shapes (spec §20.3) discriminated by the `mode` field: [`ElicitRequestFormParams`](./ElicitRequestFormParams.md) (`"form"`) and [`ElicitRequestURLParams`](./ElicitRequestURLParams.md) (`"url"`).

## Remarks

The form-mode discriminator is OPTIONAL on the wire — a request with no `mode` field MUST be treated as form mode (§20.3). Serialization therefore writes `"mode": "form"` explicitly, while a deserializer that defaults a missing discriminator to [`ElicitRequestFormParams`](./ElicitRequestFormParams.md) preserves that backwards-compatibility rule.
