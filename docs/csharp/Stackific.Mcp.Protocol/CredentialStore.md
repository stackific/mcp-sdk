# CredentialStore class

A per-authorization-server credential store keyed by `issuer`, enforcing the §23.1 isolation rules (R-23.1-i – R-23.1-l): registration state is kept separate per `issuer`, a lookup never returns another server's credentials, and [`NeedsReregistration`](./CredentialStore/NeedsReregistration.md) reports when the indicated authorization server changes so the client re-registers/re-discovers against the new one.
