# ParamHeaders.FilterValidTools&lt;TTool&gt; method

Filters a `tools/list` result, excluding only tools whose `x-mcp-header` annotations are invalid and keeping all valid tools usable. (R-9.5.1-i, R-9.5.1-j) The returned warnings name each rejected tool and the reason so the caller can log them. (R-9.5.1-k)

```csharp
public static FilterToolsResult<TTool> FilterValidTools<TTool>(IReadOnlyList<TTool> tools, 
    Func<TTool, string> nameOf, Func<TTool, JsonNode?> inputSchemaOf)
```

| parameter | description |
| --- | --- |
| TTool | The tool type. |
| tools | The tools to filter. |
| nameOf | Extracts a tool's name (for the warning). |
| inputSchemaOf | Extracts a tool's input schema (may be `null`). |

## Return Value

The valid tools and the rejection warnings.

## Remarks

Clients on non-HTTP transports MAY skip this entirely (R-9.5.1-l) — it is only invoked by the Streamable HTTP client. The tool type is generic so this works against any tool representation; the caller supplies accessors for the name and input schema.
