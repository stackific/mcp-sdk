## Remarks

Checks each annotation's name (R-9.5.1-a/b/c), that the annotated parameter's type is a primitive `integer`/`string`/`boolean` (R-9.5.1-e) and not `number` (R-9.5.1-f), and that all names are case-insensitively unique within the schema (R-9.5.1-d). Annotations at any nesting depth are accepted (R-9.5.1-h).
