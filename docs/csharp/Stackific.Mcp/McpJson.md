## Remarks

MCP field names are mostly `camelCase` (for example `supportedVersions`, `serverInfo`), so CamelCase is the default for any unannotated member; the handful of reserved, dotted keys (such as `io.modelcontextprotocol/protocolVersion`) are written verbatim via explicit JsonPropertyNameAttribute annotations.

`null` members are omitted on write (MCP treats an absent optional field and a `null` field identically), and the options are frozen so the configuration cannot drift at runtime.
