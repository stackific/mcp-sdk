# MetaKeys.IsValidKey method

Returns `true` if *key* is a syntactically valid `_meta` key AND its prefix (if present) is not reserved (§2.6.2 / §4.2, R-2.6.2-i, R-2.6.2-j).

```csharp
public static bool IsValidKey(string key)
```

| parameter | description |
| --- | --- |
| key | The candidate key. |

## Return Value

`true` when the key is well-formed and not under a reserved prefix.

## Remarks

This folds reserved-prefix rejection into validity exactly as the TypeScript `isValidMetaKey` does: a third-party key under a reserved prefix such as `io.modelcontextprotocol/something` is NOT valid for an arbitrary caller and returns `false`. The reserved bare trace-context keys (`traceparent`, `tracestate`, `baggage`) are always valid because the spec permits them. Use [`IsReservedMetaKeyPrefix`](./IsReservedMetaKeyPrefix.md) directly when you need to test reservation independently of validity.
