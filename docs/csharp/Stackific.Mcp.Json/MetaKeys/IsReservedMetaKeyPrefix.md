# MetaKeys.IsReservedMetaKeyPrefix method

Returns `true` if *prefix* is reserved — its second label is `modelcontextprotocol` or `mcp` (§2.6.2 / §4.2, R-2.6.2-f). Implementations MUST NOT define `_meta` keys under a reserved prefix except as specified by the protocol or an MCP-published extension.

```csharp
public static bool IsReservedMetaKeyPrefix(string prefix)
```

| parameter | description |
| --- | --- |
| prefix | A prefix (with or without a trailing slash) or a full key; the portion up to the first slash is examined. |

## Return Value

`true` when the second label is protocol-reserved.

## Remarks

This accepts both a bare prefix string (for example `"io.modelcontextprotocol/"`) and a full key (for example `"io.modelcontextprotocol/protocolVersion"`): the body considered for label-splitting is everything before the first slash, mirroring the TypeScript `isReservedMetaKeyPrefix` while remaining a drop-in for full keys.
