# McpClient.SubscribeAsync method

Opens a subscription stream for server-initiated change notifications (spec §10): sends `subscriptions/listen`, awaits the acknowledgement, and delivers each matching change notification to *onNotification* until the returned handle is unsubscribed.

```csharp
public Task<SubscriptionHandle> SubscribeAsync(SubscriptionFilter filter, 
    Action<JsonRpcNotification>? onNotification = null, 
    CancellationToken cancellationToken = default)
```

| parameter | description |
| --- | --- |
| filter | The notification kinds to subscribe to. |
| onNotification | Invoked for each change notification on the stream. |
| cancellationToken | Cancels opening the stream. |

## Return Value

A handle carrying the honored filter and an unsubscribe action.

## Remarks

The subscription's request-scoped lifecycle (spec §10.7) is tracked through a client-side [`SubscriptionRegistry`](../../Stackific.Mcp.Protocol/SubscriptionRegistry.md): a [`Subscription`](../../Stackific.Mcp.Protocol/Subscription.md) is created and acknowledged (`opening` → `active`) on open, routed by `io.modelcontextprotocol/subscriptionId`, and closed (with the ClientCancel reason) and removed on unsubscribe. The id is the `subscriptions/listen` request id (R-10.4-c). Inspect the registry via [`ActiveSubscriptionIds`](./ActiveSubscriptionIds.md) / [`GetSubscription`](./GetSubscription.md).
