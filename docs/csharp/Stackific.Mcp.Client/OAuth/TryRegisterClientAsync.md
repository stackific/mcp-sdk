# OAuth.TryRegisterClientAsync method

Performs dynamic client registration (spec §23.4 / RFC 7591), surfacing a structured result rather than throwing so the client can handle a registration failure and decide whether to retry with an adjusted `application_type` (R-23.4-p, R-23.4-q, R-23.4-r). The body always carries the REQUIRED `application_type`, classified from the redirect URIs.

```csharp
public static Task<DynamicClientRegistrationResult> TryRegisterClientAsync(HttpClient http, 
    JsonObject asMeta, string clientName, IReadOnlyList<string> redirectUris, 
    IReadOnlyList<string>? grantTypes = null)
```

| parameter | description |
| --- | --- |
| http | The HTTP client to use. |
| asMeta | The authorization-server metadata. |
| clientName | The human-readable client name. |
| redirectUris | The client's redirect URIs. |
| grantTypes | OPTIONAL requested grant types (e.g. include `refresh_token` for refresh; R-23.9-a). |

## Return Value

The structured registration result (success or a meaningful, possibly-retryable failure).
